<p><strong>Project Overview:</strong><br />
Hasen is a full-stack multiplayer card game featuring real-time WebSocket event architecture. The project applies Domain-Driven Design principles with a shared domain layer consumed by both the Vue 3 frontend and Node.js backend, ensuring consistency in game rules and state management.</p>

<h2>Objectives</h2>

<ol>
    <li>Build a real-time multiplayer card game with WebSocket communication</li>
    <li>Implement Domain-Driven Design with shared domain layer</li>
    <li>Create a scalable architecture supporting multiple game modes</li>
    <li>Provide smooth user experience with real-time updates</li>
</ol>

<h2>Features</h2>

<h3>1. Real-Time WebSocket Communication:</h3>

<ul>
    <li>Using Socket.IO for real-time event handling</li>
    <li>Lobby system for matchmaking and room management</li>
    <li>Turn management with synchronized game state across clients</li>
    <li>Real-time score updates and validation</li>
</ul>

<h3>2. Domain-Driven Design Architecture:</h3>

<ul>
    <li>Shared domain layer (cards, rules, events) consumed by both frontend and backend</li>
    <li>Type-safe game logic across the entire stack</li>
    <li>Consistent state management and business rules</li>
</ul>

<h3>3. User Experience:</h3>

<ul>
    <li>Responsive UI with TailwindCSS</li>
    <li>Smooth animations and transitions</li>
    <li>Mobile-first responsive layout</li>
</ul>
